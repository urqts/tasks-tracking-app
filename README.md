# TaskFlow

A personal productivity and task-management app — tasks, homework, assignments, projects, recurring tasks and goals in one clean, fast workspace. Built to run entirely on free tiers (Supabase + Vercel).

It blends ideas from Todoist, Notion, Trello and TickTick while staying simple: a list view, a Kanban board, a calendar, analytics, and reminders.

## Features

- **Auth** — register, login, logout, email verification, forgot/reset password. Protected routes via middleware.
- **Dashboard** — today / upcoming / overdue / completed-today counts, productivity score, weekly progress, completion trend, priority breakdown, project progress.
- **Tasks** — full CRUD with title, description, priority (Low/Medium/High/Urgent), status (Todo/In Progress/Review/Completed/Archived), start & due dates, estimated/actual time, category, project, tags. Multi-select, bulk complete/delete, quick-complete, search, sort, filter.
- **Subtasks** — nested subtasks with automatic parent progress.
- **Kanban board** — drag tasks across columns (native HTML5 DnD) with optimistic updates.
- **Calendar** — month view, drag-to-reschedule, click-to-edit.
- **Projects** — color, deadline, computed progress and task counts.
- **Categories & tags** — custom, color-coded.
- **Recurring tasks** — daily / weekly / monthly / yearly / custom intervals; future tasks generated by a cron job.
- **Attachments** — upload any file type to Supabase Storage, preview/download via signed URLs, delete.
- **Notifications** — browser notifications for due/overdue tasks; email digests via the cron endpoint.
- **Analytics** — completion trends, category/priority/status distributions, productivity score (Recharts).
- **Export** — Excel (.xlsx) and CSV for all / filtered / completed tasks (SheetJS).
- **Settings** — theme (light/dark/system), default view, timezone, language, notification preferences.
- **UX** — fully responsive (mobile-first), loading skeletons, empty states, error boundaries, toasts.

## Quick start

1. **Install dependencies**

   ```bash
   npm install
   ```

2. **Create a Supabase project** at https://supabase.com (free tier).

3. **Configure environment** — copy `.env.example` to `.env` and fill in the values (see comments in that file). You need the project URL, anon key, service-role key, and both the pooled `DATABASE_URL` and direct `DIRECT_URL` connection strings.

4. **Create the database schema**

   ```bash
   npx prisma migrate dev --name init
   # or, to push without migration history:
   npx prisma db push
   ```

5. **Create the Storage bucket** — in the Supabase dashboard, Storage → New bucket → name it `attachments` (matching `NEXT_PUBLIC_SUPABASE_STORAGE_BUCKET`). Keep it private; the app generates signed URLs.

6. **Run the app**

   ```bash
   npm run dev
   ```

   Open http://localhost:3000, register an account, and you're in.

> **Email verification:** by default Supabase requires email confirmation. For local testing you can disable it under Authentication → Providers → Email, or use the link from the Supabase Auth logs.

## How key pieces fit together

- **Auth & data join.** Supabase owns identity; `lib/auth.ts` (`ensureUser`) upserts a matching row in our `users` table (same UUID) on first login and lazily creates default `Settings`. Every server action calls `requireUserId()` so all queries are scoped to the signed-in user.
- **Server Actions.** All mutations live in `actions/*` and return a typed `ActionResult`. Inputs are validated with Zod (`lib/validations/*`) on the server. Pages revalidate automatically.
- **Soft deletes.** Models carry `deletedAt`; deletes set it and all reads filter `deletedAt: null`.
- **Recurring tasks & email.** `app/api/cron` materializes due recurrences and sends email digests. Schedule it with Vercel Cron (`vercel.json`) and protect it with `CRON_SECRET`.
- **Attachments.** Files upload straight from the browser to Supabase Storage; only metadata round-trips through a server action. Downloads use short-lived signed URLs.

See `DEPLOYMENT.md` for production setup on Vercel + Supabase, including cron and email.

## Security notes

- All inputs validated with Zod server-side; every query scoped to the authenticated user.
- The Supabase **service-role key** is used only in `lib/supabase/server.ts#createServiceClient` and the cron route — never imported into client code.
- The cron endpoint is gated by a bearer `CRON_SECRET`.
- Recommended hardening: enable **Row Level Security** policies in Supabase as defense-in-depth (the app already scopes by `userId`, but RLS protects the DB directly). See `DEPLOYMENT.md`.
